Major changes

  • Triggering condition: You can enter a different kind of condition to determine whether the action should be triggered.
  • Gesture training adjustment: Add new gesture in the action dialog.
  • You can change the threshold of blocking touch input and drawing gesture.
  • Solve drawing failure problem under certain condition.

Triggering Condition

Concatenation is allowed using Boolean AND, OR, and NOT operators. You can use parentheses to group clauses and force precedence. The AND operator has precedence over other operators. For example:
finger_1_ID>finger_2_ID and finger_1_ID>finger_3_ID

When you create comparison expressions, the following operators are allowed:

<

>

<=

>=

<>

=

The following arithmetic operators are also supported in expressions:

+ (addition)

- (subtraction)

* (multiplication)

/ (division)
